# Парсинг ссылок в VK боте Петька

## Описание

Реализована функциональность автоматического парсинга и чтения ссылок из сообщений VK. Бот теперь может:
- Автоматически обнаруживать URL в сообщениях
- Загружать содержимое ссылок
- Извлекать заголовки, описания и основной текст страниц
- Использовать эту информацию для формирования контекстных ответов

## Возможности

### Поддерживаемые форматы URL
- Полные URL с протоколом: `https://example.com/page`
- Короткие URL без протокола: `vk.com/wall123`, `t.me/channel`
- Популярные платформы: VK, Telegram, YouTube, GitHub, Habr и др.
- URL с параметрами: `https://site.com/page?id=123&lang=ru`

### Режимы работы

#### 1. Базовый режим (link_parser.py)
- Использует только стандартную библиотеку Python и `requests`
- Парсинг HTML с помощью регулярных выражений
- Минимальные зависимости
- Подходит для простых случаев

**Зависимости:**
```bash
pip install requests
```

#### 2. Расширенный режим (link_parser_enhanced.py)
- Использует BeautifulSoup для более качественного парсинга
- Поддержка Open Graph метатегов
- Интеллектуальное извлечение основного контента
- Автоматический fallback на базовый режим

**Зависимости:**
```bash
pip install requests beautifulsoup4 lxml
```

### Извлекаемая информация
- **Заголовок** страницы (title, og:title)
- **Описание** (meta description, og:description)
- **Основной текст** (автоматическое определение контентной зоны)
- **Изображение** (og:image) - в расширенном режиме
- **Финальный URL** (после редиректов)

## Использование

### Автоматическая обработка в боте

Бот автоматически обнаруживает и обрабатывает ссылки в сообщениях:

**Пример 1 - Простой запрос:**
```
Пользователь: Петька, что тут написано? https://example.com/article
Бот: [загружает содержимое страницы и отвечает на основе текста]
```

**Пример 2 - Множественные ссылки:**
```
Пользователь: Сравни эти статьи:
  https://habr.com/post/123
  https://github.com/user/repo
Бот: [загружает обе ссылки и сравнивает их содержимое]
```

**Пример 3 - Ссылки без протокола:**
```
Пользователь: Петя, посмотри vk.com/wall-123_456
Бот: [автоматически добавляет https:// и обрабатывает]
```

### Программное использование

#### Базовый парсер

```python
from link_parser import LinkParser, extract_and_fetch_links

# Создание парсера
parser = LinkParser()

# Извлечение URL из текста
urls = parser.extract_urls("Посмотри эту ссылку: https://example.com")
print(urls)  # ['https://example.com']

# Загрузка содержимого ссылки
result = parser.fetch_link_content("https://example.com")
if result['success']:
    print(f"Заголовок: {result['title']}")
    print(f"Описание: {result['description']}")
    print(f"Текст: {result['text'][:200]}...")

# Быстрая обработка сообщения
message = "Петька, что тут? https://example.com"
formatted = extract_and_fetch_links(message, max_links=3)
print(formatted)
```

#### Расширенный парсер

```python
from link_parser_enhanced import EnhancedLinkParser, get_links_context_for_gpt

# Создание расширенного парсера
parser = EnhancedLinkParser()

# Извлечение и форматирование для пользователя
message = "Проверь эти ссылки: https://habr.com/post/123 и github.com/user/repo"
formatted = parser.format_links_context(
    parser.extract_urls(message),
    max_links=3
)

# Получение контекста для GPT
gpt_context = get_links_context_for_gpt(message, max_links=3)
```

## Настройки и ограничения

### Параметры парсера

```python
parser = EnhancedLinkParser(
    timeout=10,  # Таймаут HTTP запросов (секунды)
    user_agent="..."  # Custom User-Agent
)
```

### Ограничения

- **Максимум ссылок:** Обрабатывается до 3 ссылок из одного сообщения (настраивается)
- **Таймаут:** 10 секунд на загрузку каждой ссылки
- **Длина текста:** До 5000 символов извлекаемого текста (настраивается)
- **Задержка:** 0.5 секунды между запросами (для соблюдения rate limits)

### Обработка ошибок

Парсер gracefully обрабатывает:
- Таймауты при загрузке
- Недоступные страницы (404, 403, и т.д.)
- Некорректный HTML
- Отсутствие интернет-соединения

В случае ошибки бот продолжает работу без парсинга ссылок.

### Поток обработки

1. **Получение сообщения** → Бот получает сообщение из VK
2. **Извлечение URL** → `LinkParser.extract_urls()` находит все ссылки
3. **Загрузка содержимого** → HTTP запрос к каждой ссылке (до 3)
4. **Парсинг HTML** → Извлечение title, description, text
5. **Формирование контекста** → Добавление в контекст для GPT
6. **Генерация ответа** → YandexGPT использует контекст из ссылок

### Методы класса EnhancedLinkParser

| Метод | Описание |
|-------|----------|
| `extract_urls(text)` | Извлекает все URL из текста |
| `fetch_link_content(url)` | Загружает и парсит одну ссылку |
| `fetch_multiple_links(urls)` | Загружает несколько ссылок с задержкой |
| `format_link_info(link_info)` | Форматирует информацию для вывода |
| `format_links_context(urls)` | Создает контекст для GPT |

## Отладка

### Логирование

При обработке ссылок бот выводит информацию:

```
📎 Найдено ссылок в сообщении: 2
  - https://example.com/page1
  - https://example.com/page2
```

### Проблемы и решения

**Проблема:** Ссылки не обрабатываются
- Проверьте установлен ли `requests`: `pip list | grep requests`
- Проверьте логи бота на наличие ошибок

**Проблема:** Плохое качество парсинга
- Установите BeautifulSoup: `pip install beautifulsoup4 lxml`
- Расширенный парсер даст лучшие результаты

**Проблема:** Таймауты
- Увеличьте таймаут: `LinkParser(timeout=20)`
- Уменьшите количество обрабатываемых ссылок
