## Взаимодействие всех функций (словесное объяснение):

### 📋 **Общая архитектура "заводской конвейер":**

Представьте, что бот - это завод по обработке сообщений.

---

### 🏭 **ЦЕХ 1: Прием сырья (получение сообщений)**
**Функция:** `check_new_messages()`
- **Что делает:** Постоянно проверяет "почтовый ящик" на новые непрочитанные сообщения
- **Как работает:** Каждые 3 секунды заглядывает в VK и собирает новые письма
- **Результат:** Передает сырые сообщения на следующий цех

---

### 🚚 **ЦЕХ 2: Сортировка и фильтрация**
**Функции:** `is_private_message()`, `is_triggered_message()`
- **Что делает:** Сортирует сообщения:
  - Личные сообщения → обрабатываем ВСЕ
  - Беседы → обрабатываем только если есть триггеры
- **Как работает:** 
  - Смотрит на ID отправителя (peer_id)
  - Ищет ключевые слова ("Петька", "Петя" и т.д.)
  - Проверяет reply/forward сообщения
- **Результат:** Отсеивает спам, оставляет только важные сообщения

---

### 🧹 **ЦЕХ 3: Очистка и подготовка**
**Функции:** `clean_question()`, `extract_target_message()`
- **Что делает:** Подготавливает сообщение к обработке
- **Как работает:**
  - Удаляет триггерные слова из текста
  - Если текст стал пустым → генерирует стандартный вопрос
  - Извлекает reply/forward сообщения
- **Результат:** Чистый, готовый к анализу вопрос

---

### 📚 **ЦЕХ 4: Поиск контекста**
**Функции:** `get_conversation_context()`, `process_forward_messages()`
- **Что делает:** Ищет историю диалога для понимания контекста
- **Как работает:**
  - Достает последние 10 сообщений из архива этой беседы
  - Форматирует имена пользователей через `get_user_info()`
  - Обрабатывает пересланные сообщения
- **Результат:** Полная картина того, о чем шла речь ранее

---

### 🧠 **ЦЕХ 5: Генерация ответа (мозг завода)**
**Функция:** `ask_yandex_gpt()`
- **Что делает:** Превращает вопрос+контекст в умный ответ
- **Как работает:**
  - Отправляет запрос к Yandex GPT API
  - Использует разные "инструкции" для ЛС и бесед
  - Получает сгенерированный текст
- **Результат:** Готовый, осмысленный ответ

---

### 📤 **ЦЕХ 6: Отправка продукции**
**Функция:** `send_message()`
- **Что делает:** Доставляет готовый ответ пользователю
- **Как работает:** Отправляет сообщение через VK API
- **Результат:** Пользователь получает ответ

---

### 📂 **СКЛАД: Хранение истории**
**Класс:** `ConversationManager`
- **Что делает:** Ведет архив всех бесед
- **Как работает:**
  - Каждая беседа/ЛС → отдельный файл
  - Новые сообщения добавляются в конец
  - Автоматически загружает всю историю при первом обращении
- **Результат:** Постоянная память о всех диалогах

---

### 🔄 **СЛУЖБА ОБНОВЛЕНИЯ: Техническое обслуживание**
**Класс:** `TokenUpdater`
- **Что делает:** Меняет "ключи доступа" к Yandex GPT каждые 12 часов
- **Как работает:** Автоматически обновляет токен в фоновом режиме
- **Результат:** Бот всегда имеет рабочий доступ к AI

---

### 🎯 **Пример полного цикла обработки:**

1. **Пользователь в беседе пишет:** "Петька, как дела?"
2. **Цех 1:** `check_new_messages()` обнаруживает новое сообщение
3. **Цех 2:** `is_triggered_message()` видит "Петька" → пропускает дальше
4. **Цех 3:** `clean_question()` удаляет "Петька," → остается "как дела?"
5. **Цех 4:** `get_conversation_context()` находит последние сообщения этой беседы
6. **Цех 5:** `ask_yandex_gpt()` получает: "Контекст + как дела?" → генерирует ответ
7. **Цех 6:** `send_message()` отправляет ответ в беседу
8. **Склад:** `ConversationManager` сохраняет и вопрос, и ответ в историю

**Результат:** Пользователь получает осмысленный ответ, учитывающий контекст беседы! 🎉
